# node-app

[![Version](https://img.shields.io/npm/v/@donniean/node-app.svg)](https://www.npmjs.com/package/@donniean/node-app) [![License: MIT](https://img.shields.io/github/license/donniean/node-app)](https://github.com/donniean/node-app/blob/main/LICENSE) [![CI](https://github.com/donniean/node-app/actions/workflows/ci.yaml/badge.svg)](https://github.com/donniean/node-app/actions/workflows/ci.yaml) [![Release](https://github.com/donniean/node-app/actions/workflows/release.yaml/badge.svg)](https://github.com/donniean/node-app/actions/workflows/release.yaml)

A public template repository for personal use. It provides reusable Node.js CLI, Node.js API, and npm package starters, templates, examples, demos, and scaffolds. Projects created from this template are not limited to personal use and may be adapted for individual, team, or company use.

The repository also contains a buildable and publishable example npm package, [`@donniean/node-app`](package.json), and provides the `configs-md` CLI. The CLI generates Markdown documentation from configuration definitions in [`src/configs/`](src/configs/) for documenting setup and cleanup commands for common project tools.

## Contents

- [`.changeset/`](.changeset/): Changesets release configuration.
- [`bin/cli.js`](bin/cli.js): Executable wrapper included in the published package; it points to the build output `dist/cli.js`.
- [`src/cli.ts`](src/cli.ts): Commander CLI entry for `configs-md`.
- [`src/api.ts`](src/api.ts): Markdown generation API that exports `getMarkdown`, `writeMarkdown`, and `writeMarkdownWithDefaults`.
- [`src/index.ts`](src/index.ts): Package export entry point, currently exporting [`src/api.ts`](src/api.ts).
- [`src/configs/`](src/configs/): Tool configuration definitions and the main source for generating [`configs.md`](configs.md).
- [`src/models/`](src/models/), [`src/helpers/`](src/helpers/), [`src/utils/`](src/utils/): Schemas, command builders, path resolution, package metadata reading, and shared utilities.
- [`configs.md`](configs.md): Configuration documentation generated by `pnpm run docs`; this is not a hand-written `docs/**/*.md` document.

When this template is used to create an actual project, replace, trim, or delete the sample CLI, API, configuration generation logic, and placeholder structure according to the project goal. The current directories and the `configs-md` example describe this template's current implementation, not conventions that every downstream project must keep.

## Requirements

- The Node.js version is defined by [`.nvmrc`](.nvmrc) and `engines.node` in [`package.json`](package.json).
- The pnpm version is defined by `packageManager` and `engines.pnpm` in [`package.json`](package.json).
- Use `pnpm`. `preinstall` enforces this with `only-allow`.

## Commands

```bash
pnpm install
pnpm run build
pnpm run docs
pnpm run quality:check
pnpm run quality:fix
pnpm run test
```

Debug the CLI locally:

```bash
pnpm run dev --help
pnpm run dev --file configs.md
```

Published entry points:

- CLI: `configs-md`
- API: `@donniean/node-app`

## Generated Documentation

[`configs.md`](configs.md) is generated by `pnpm run docs`. To update the configuration documentation, prefer changing the configuration definitions in [`src/configs/`](src/configs/), then run:

```bash
pnpm run docs
```

The default output file name and `files.download` base URL are defined in [`src/models/configs.constants.ts`](src/models/configs.constants.ts).

When changing [`src/configs/`](src/configs/), [`src/helpers/`](src/helpers/), [`src/models/`](src/models/), [`src/scripts.ts`](src/scripts.ts), or [`src/api.ts`](src/api.ts), check whether [`configs.md`](configs.md) needs to be regenerated.

## Release

- [`ci.yaml`](.github/workflows/ci.yaml) runs quality checks, test, and build on pull requests to `main` and manual workflow dispatch.
- [`release.yaml`](.github/workflows/release.yaml) runs build after pushes to `main`, then uses `changesets/action` to create a release pull request or publish to npm.
- [`dependencies.bump.yaml`](.github/workflows/dependencies.bump.yaml), [`pull-requests.auto-merge.yaml`](.github/workflows/pull-requests.auto-merge.yaml), and [`pull-requests.auto-update.yaml`](.github/workflows/pull-requests.auto-update.yaml) reuse workflows from [`donniean/hub`](https://github.com/donniean/hub).
- [`.github/dependabot.yaml`](.github/dependabot.yaml) enables daily updates for npm and GitHub Actions dependencies.

This repository uses Changesets for versioning and releases. Use the existing scripts when maintaining releases manually. Before running release-related commands, confirm the npm package, credentials, target version, and release intent:

```bash
pnpm run changeset:add
pnpm run changeset:version
pnpm run changeset:publish
```
